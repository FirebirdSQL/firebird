Firebird Database Server 1.5.5
==============================

Este documento es una guía para instalar este paquete
de Firebird 1.5 en la plataforma Win32. Estas notas
hacen referencia al paquete de instalación en sí, más
que a Firebird 1.5 en general.

Antes de instalar
-----------------

Se recomienda que DESINSTALE cualquier versión
existente de Firebird 1.0, Firebird 1.5, Firebird 2.0
o InterBase antes de instalar este paquete.


Instalación en sistemas con Firebird 2.0
----------------------------------------
El proceso de instalación de Firebird 1.5.5 
detectará una instalación de Firebird 2. En este caso, 
presentará una advertencia y le permitirá cancelar la instalación.
Si continúa, el proceso de instalación detectará una instancia
de Super Server en ejecución, y la instalación será cancelada.
No obstante, no podrá detectar un Classic Server en ejecución,
y continuará la instalación. Por esta razón es recomendable
verificar manualmente el tipo de servidor utilizado si el 
instalador advierte sobre una instalación existente.


Nuevas características del instalador en Firebird 1.5.4
-------------------------------------------------------

o Si se encuentra el archivo firebird.conf el 
  instalador instalará el archivo de configuración
  por defecto como:

  firebird.conf.default.

  El archivo firebird.conf existente no se modificará.
  Esto busca asegurar que las restricciones de
  acceso se mantengan luego de una actualización.


o Nuevos lenguajes han sido incorporados al instalador
  - Bosnio, Español, Italiano, Polaco y Ruso.
  
  Nota 1 - Los lenguajes se listan ordenados alfabéticamente
  por su nombre original, no por su nombre en inglés.
  
  Nota 2 - Los lenguajes disponibles están determinados
  por las páginas de códigos instaladas. Los lenguajes del
  Este de Europa no estarán disponibles para usuarios del 
  Oeste de Europa.


Nuevas características del instalador en Firebird 1.5.3
-------------------------------------------------------
  
El instalador soporta ahora varios lenguajes, pero SOLO
durante el proceso de instalación en sí.


Nuevas características del instalador en Firebird 1.5
-----------------------------------------------------

o Este instalador ahora combina los ejecutables de
  super server y classic server en un mismo paquete
  de instalación. Ud. puede elegir instalar uno o el
  otro, pero no ambos. Para cambiar de tipo de
  servidor deberá desinstalar y re-instalar.
 
o Las reglas para la instalación de librerías han
  cambiado considerablemente. Están explicadas en
  detalle en el archivo.\docs\README.Win32LibraryInstallation.txt,
  que estará disponible al finalizar la instalación.

  Como resultado de estas nuevas reglas el instalador
  verifica la existencia de una instalación previa
  de Firebird o Interbase
 
  - Si Firebird 1.5 está instalado, tratará de
    sobrescribir la instalación. Si el servidor
    se está ejecutando, se detendrá la instalación.
   
  - Si se encuentra otra versión de Firebird o
    Interbase instalada, se notificará al usuario.
    Si el usuario continúa, el proceso instalará
    Firebird y creará las entradas de registro
    pero no configurará Firebird para ejecutarse,
    ni como servicio ni como aplicación. Esto
    debe hacerse manualmente.

  - El instalador tiene una nueva opción de línea
    de comando
      /force
    que les permite a los que tienen una actitud
    del tipo 'que me importa' no tomar en cuenta
    la regla anterior.


Características obsoletas relacionadas con la instalación
---------------------------------------------------------

o Firebird 1.0 reservaba una clave de registro para
  uso de Firebird:
 
    HKLM\SOFTWARE\FirebirdSQL
  Esto ya no es válido y será borrada por el
  instalador. Si Ud. tiene alguna aplicación que
  depende de esta clave, debería volver a agregarla
  manualmente luego de la instalación. No obstante, es preferible
  que actualice su aplicación para que lea la nueva clave.
 
o Las primeras versiones pre-lanzamiento de
  Firebird 1.5 instalaban fbclient.dll en el
  directorio <system>. Esta práctica ya es obsoleta.
  Se ofrece ahora una opción para copiar
  la librería en el directorio <system> en el
  momento de la instalación; no obstante, es
  preferible que actualice sus aplicaciones
  en conformidad con el nuevo uso de fbclient.


Desinstalación
--------------

o Es recomendable que este paquete sea correctamente
  desinstalado utilizando el desinstalador provisto, 
  que puede ser utilizado desde el Panel de Control.
  Como alternativa, puede ser desinstalado ejecutando
  unins000.exe directamente desde la carpeta de 
  instalación.

o Si Firebird se está ejecutando como una aplicación
  (en lugar de hacerlo como servicio), se recomienda
  detener el servidor manualmente antes de ejecutar
  el desinstalador, debido a que este último no puede
  detener una aplicación en ejecución.
  Si el servidor estuviera ejecutándose durante el 
  proceso de desinstalación, éste no se completará
  correctamente. Deberá eliminar los remanentes
  manualmente.
 

Otras notas
-----------

  Firebird requiere WinSock2. Todas las plataformas
  Win32 deberían tenerlo, con excepción de Win95. 
  Durante la instalación se realiza una verificación
  de la librería Winsock2. Si la misma no se encuentra,
  la instalación fallará. Puede visitar el siguiente
  enlace:

    http://support.microsoft.com/default.aspx?scid=kb;EN-US;q177719

  para obtener información sobre cómo realizar la 
  actualización.


Instalación desde un archivo batch
----------------------------------

El programa de instalación puede ser ejecutado desde
un archivo batch. Se pueden utilizar los siguientes 
parámetros:

/SP-
  Deshabilita el mensaje 'Este programa instalará... ¿Desea
  continuar?' al comienzo de la instalación.

/SILENT, /VERYSILENT
  Configura el instalador para ser silencioso o muy
  silencioso. Cuando la instalación es silenciosa ni el
  asistente ni la ventana de fondo se muestran pero
  la ventana de progreso de instalación sí. Cuando
  la instalación es muy silenciosa tampoco se muestra
  esta ventana de progreso. Todo lo demás, por ejemplo
  el mensaje inicial (si no se ha usado el parámetro
  '/SP-') y los mensajes de error durante la
  instalación, se muestra normalmente.

  Si es necesario un reinicio y no se usa el parámetro
  '/NORESTART' (ver más abajo) y la instalación es
  silenciosa, se mostrará un cuadro de mensaje
  '¿Reiniciar ahora?'. Si está en modo
  muy silencioso, reiniciará sin preguntar.


/NORESTART
  Indica al instalador que no debe reiniciar, aún si
  es necesario.
 
/DIR="x:\directorio"
  Reemplaza el directorio por defecto en la página
  de selección de directorio de destino del asistente.
  Se debe especificar una ruta completa incluyendo la unidad.
  Si la directiva DisableDirPage de la sección [Setup] 
  está seteada como 'yes', este parámetro de línea
  de comando se ignora.
 
/GROUP="Nombre de Carpeta"
  Reemplaza el nombre de carpeta por defecto mostrado
  en la página de selección de carpeta del Menú Inicio
  del asistente. Si la directiva DisableProgramGroupPage
  de la sección [Setup] se setea como 'yes', este parámetro
  se ignora.

/NOICONS
  Indica al instalador que deshabilite la opción
  'No crear iconos' en la página de selección
  de carpeta del Menú Inicio del asistente.

/COMPONENTS="lista de componentes separados por comas"

  Los componentes pueden ser
                ServerComponent,
                ServerComponent\SuperServerComponent, 
                ServerComponent\ClassicServerComponent,
                DevAdminComponent y 
                ClientComponent

  Reemplaza la configuración de componentes por
  defecto. Si se usa este parámetro la instalación 
  selecciona automáticamente el tipo 'personalizado'. 
  Una instalación completa requiere combinar 
  componentes.
  Por ejemplo:
 
  /COMPONENTS="ServerComponent, 
               ServerComponent\SuperServerComponent,  
               DevAdminComponent, ClientComponent"
 
  sería lo necesario para una instalación completa.


/FORCE
  Indica al instalador que ignore su análisis del
  entorno existente. Intentará instalar y configurar
  Firebird 1.5 como si no hubiera una versión previa
  de Firebird o Interbase instalada.

  Esto puede ser útil si Ud. tiene una instalación
  muy dañada que no quiere desinstalar. O puede ser
  otra forma de hacer enojar a sus usuarios,
  rompiendo una instalación previa perfectamente
  funcional de Interbase. Es su opción.


/NOCPL
  No instalar el Applet del Panel de Control. Esto
  es útil por dos razones:
 
  o Instalar/desinstalar el applet del Panel de
    Control frecuentemente requiere un reinicio
    del sistema.

  o Ud. puede desear usar un applet alternativo.

 
/NOGDS32
  No instalar una copia de la librería cliente
  en el directorio de sistema, aún si el análisis
  de instalación concluye que es correcto hacerlo.


/COPYFBCLIENT
  Copia fbclient.dll al directorio de sistema. Esto
  se recomienda para instalaciones de cliente si
  Ud. está seguro que solamente usará una única
  versión de servidor. Si espera que su aplicación
  pueda acceder a diferentes versiones de servidor,
  no se recomienda este parámetro. Vea
 
    doc/README.Win32LibraryInstallation.txt

  para más información.
