==================================================
Сервер Баз Данных Firebird 2.0.2
==================================================

  ** ВАЖНО **

  Изменилась ODS по сравнению с Firebird 1.5. Смотрите
  примечания ниже относительно новой ODS и установки
  поверх старой версии Firebird.

  ** ***** **

Этот документ является руководством по установке
пакета Firebird 2.0.2 на платформе Win32. Эти
замечания относятся как к установке пакета, так и к
самому Firebird 2.0.2 в целом.


Содержание
----------

o Перед установкой
o Новая ODS 11.0
o Установка на систему с Firebird 1.5
o Установка на систему с Firebird 2.0
o Новые возможности инсталлятора
o Известные проблемы при инсталляции
o Деинсталляция
o Другие примечания
o Установка из пакетного (bat) файла


Перед установкой
----------------

Рекомендуется перед установкой этого пакета
ДЕИНСТАЛЛИРОВАТЬ все предыдущие версии Firebird или
InterBase. Особенно важно убедиться, что fbclient.dll
и gds32.dll удалены из каталога <system32>.


Новая ODS 11.0
--------------

Особенностью Firebird 2.0 является новая ODS 11.0,
которая полностью не совместима с ODS 11.0, которая
использовалась предыдущими альфа релизами.

Если вы устанавливаете этот релиз Firebird 2.0 в
первый раз поверх предыдущей установки Firebird 2
alpha, рекомендуем вам производить установку в новый
(пустой) каталог. Это необходимо по той причине, что
в процессе деинсталляции сохраняется существующая база
данных security2. Однако, эта старая версия использует
несовместимую ODS, так что вы не сможете подключиться
к серверу.


Установка на систему с Firebird 1.5
-----------------------------------

База данных паролей была изменена. Новая база данных
называется security2.fdb и использует более устойчивый
алгоритм шифрования паролей. Более подробно о новых
возможностях относительно безопасности вы можете
узнать из release notes.

Новая структура базы данных паролей и изменения метода
шифрования создает некоторые проблемы перехода в том
случае, если у вас есть много идентификаторов
пользователей. Инсталлятор не предпринимает попыток
совершить эту миграцию. Это нужно сделать в ручную.
Возможную технику миграции вы можете найти в
misc/upgrade/security/.


Установка на систему с Firebird 2.0
-----------------------------------

Кроме изменения ODS, пожалуйста, имейте в виду, что
инсталлятор не может автоматически определить, что у
вас запущен сервера Firebird 2.0 Alpha или Beta в
качестве службы (сервиса). Поэтому вы не получите
никаких предупреждающих сообщений до тех пор, пока
инсталлятор не попробует переписать файл поверх
запущенного. Это можно предотвратить, если вы сначала
выполните деинсталляцию.


Новые возможности инсталлятора
------------------------------

o Если существует файл firebird.conf, инсталлятор
  создаст файл конфигурации по умолчанию под именем:

  firebird.conf.default.

  Существующий файл firebird.conf останется не
  тронутым. Это сделано для того, чтобы быть
  уверенным, что ограничения доступа сохранятся при
  переходе к новой версии.


o В инсталлятор добавлена возможность поддержки
  нескольких языков. Эта поддержка распространяется
  ТОЛЬКО на процесс установки.

  ПРИМЕЧАНИЕ: Поддерживаемые языки определяются
  локализацией вашей версии Windows. Это значит,
  например, что пользователи из Западной Европы
  в обычном состоянии не смогут видеть поддержку
  славянских языков.


Известные проблемы при инсталляции
----------------------------------

В настоящее время таких нет.


Деинсталляция
-------------

o Предпочтительно, чтобы этот пакет был
  деинсталлирован корректно с помощью прилагающегося
  приложения для деинсталляции. Оно может быть вызвано
  из Панели Управления (Control Panel). Другой способ:
  вы можете выполнить деинсталляцию путем прямого
  запуска файла unins000.exe из директории установки.

o Если Firebird запущен в качестве приложения (а не
  как служба), рекомендуется в ручную остановить
  сервер прежде, чем запускать деинсталлятор. Это
  необходимо выполнить потому, что деинсталлятор не
  может останавливать запущенные приложения. Если
  сервер будет работать во время деинсталляции,
  процесс деинсталляции завершиться с ошибкой. Вы
  должны будете удалить остатки вручную.

o Деинсталлятор оставляет четыре файла в каталоге
  установки:

  - aliases.conf
  - firebird.conf
  - firebird.log
  - security2.fdb

  Это происходит намеренно. Эти файлы потенциально
  могут быть изменены пользователем и могут
  потребоваться при повторной установке Firebird в
  будущем. Они могут быть удалены, если они более не
  требуются.


Другие примечания
-----------------

  Для Firebird требуется WinSock2. Все платформы Win32
  должны иметь это, за исключением Win95. Во время
  инсталляции выполняется проверка на существование
  библиотеки Winsock2. Если в процессе проверки
  библиотека не будет обнаружена, инсталляция
  завершиться неудачей. Вы можете посетить страницу:

	http://support.microsoft.com/default.aspx?scid=kb;EN-US;q177719

  чтобы найти, как выполнить обновление.


Установка из пакетного (bat) файла
----------------------------------

Программа установки может быть запущена из пакетного
файла. Пожалуйста, посмотрите документ:

	installation_scripted.txt

чтобы узнать об этом более подробно.
